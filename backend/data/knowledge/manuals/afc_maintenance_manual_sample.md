# AFC 闸机维护手册（示例）

> ⚠️ 本文件为演示用示例手册，内容不代表真实官方维修规程。
> 仅供系统功能演示和开发测试使用。

---

## 一、票卡不接收检查步骤

**故障现象**：设备无法正常接收单程票/储值卡，显示"票卡无效"或"请重新刷卡"。

**可能原因**：
1. 票卡通道有异物或灰尘堆积
2. 票箱已满或安装不到位
3. 读写器天线信号衰减
4. 票卡识别传感器灵敏度下降

**检查步骤**：
1. 打开设备前门，检查票卡通道是否有异物、灰尘或磨损痕迹
2. 确认票箱安装状态，检查票箱容量和回收机构是否正常
3. 使用测试票卡进行多次刷卡测试，观察读写器响应指示灯
4. 检查票卡通道传感器是否正常触发（可通过后台日志查看触发计数）
5. 用清洁布擦拭票卡通道和传感器表面

**备件准备**：
- 票卡通道组件
- 读写器天线模块
- 票卡通道传感器

## 二、扇门异常检查步骤

**故障现象**：扇门无法正常开合、开合速度慢、有异响、或验票后不放行。

**可能原因**：
1. 扇门机构存在机械卡滞或异物阻挡
2. 扇门电机驱动力矩不足
3. 通行检测传感器被遮挡或失效
4. 扇门控制板信号异常

**检查步骤**：
1. 目视检查扇门通道内是否有异物、垃圾或障碍物
2. 手动推动扇门，感受阻尼是否正常，判断是否存在机械卡滞
3. 检查扇门电机驱动电压和电流是否在正常范围
4. 检查通行检测传感器（红外对射）是否被遮挡或偏移
5. 通过维护终端进行扇门开合测试，记录动作时间
6. 检查扇门控制板指示灯状态和通信连接

**备件准备**：
- 扇门机构组件
- 扇门电机
- 通行检测传感器
- 扇门控制板

## 三、通信异常检查步骤

**故障现象**：设备离线、通信中断、数据上传失败、远程控制无响应。

**可能原因**：
1. 网络链路不稳定或网线松动
2. 通信模块故障
3. 交换机端口故障或网络配置错误
4. 设备IP地址冲突

**检查步骤**：
1. 检查设备网络接口指示灯状态，确认物理连接正常
2. 使用网线测试仪检查网线连通性
3. 通过ping命令测试设备IP可达性
4. 检查交换机对应端口的MAC地址学习和流量统计
5. 重启通信模块，观察是否能恢复连接
6. 检查后台系统是否存在设备离线日志记录
7. 确认设备IP、网关、子网掩码等网络参数配置正确

**备件准备**：
- 通信模块
- 网线（超五类/六类）
- 交换机端口相关备件

## 四、黑屏/死机检查步骤

**故障现象**：设备屏幕无显示、黑屏、系统死机、无响应。

**可能原因**：
1. 主控单元运行异常或程序崩溃
2. 系统软件或固件版本问题
3. 电源输出不稳定
4. 显示屏背光或驱动模块故障

**检查步骤**：
1. 检查设备电源指示灯是否正常亮起
2. 测量电源模块输出电压是否符合规格（5V/12V/24V等）
3. 检查显示屏背光是否亮起（可在暗光环境下观察）
4. 尝试通过维护终端或远程方式进行系统重启
5. 查看系统日志，检查是否有异常重启记录或错误日志
6. 检查显示屏连接线是否松动或破损
7. 如果重启后问题复现，建议更换主控板进行测试

**备件准备**：
- 主控板
- 显示屏模块
- 电源模块
- 显示屏连接线

## 五、暂停服务检查步骤

**故障现象**：设备自动进入暂停服务状态，无法恢复运营。

**可能原因**：
1. 设备状态模块检测到异常并触发暂停
2. 控制模块或软件检测到关键模块故障
3. 电源、通信或外设异常触发保护机制
4. 连续多次故障触发自我保护

**检查步骤**：
1. 查看设备本地状态指示灯和错误代码显示
2. 通过维护终端读取设备状态码和故障日志
3. 逐个检查关键模块（电源、通信、主控、票卡、扇门）状态
4. 确认是否存在连续重复的暂停服务记录（工单系统查询）
5. 尝试进行模块复位或系统重启恢复
6. 如果无法恢复，按照错误代码定位具体故障模块
7. 恢复后持续观察 24 小时，确认不再复发

**备件准备**：
- 主控模块
- 电源模块
- 通信模块
- 根据错误代码对应的模块准备相应备件

---

## 运维注意事项

1. **安全第一**：所有检查和维修操作必须在断电或安全模式下进行
2. **记录完整**：每次检查维修需详细记录现象、操作、结果和更换部件
3. **备件管理**：建议对高频故障模块保持安全库存
4. **预防性维护**：建议每季度对闸机进行全面检查和清洁
5. **软件更新**：关注设备厂商发布的固件和软件更新
